## 슬라이드 2 — 작업 목표 / 산출물

> 화면: 좌측 목표 박스, 우측 산출물 박스

**멘트**
> 목표는 단순합니다. 시중 PDF 한 회분을 통째로 우리 양식에 맞게 옮기는 것.
>
> 우리 양식이 발문·보기·해설·오답분석·학습 POINT 다섯 영역을 컬럼 단위로 나누고, 텍스트 안에는 HTML을 허용해서 수식이나 표가 바로 렌더링되도록 설계되어 있어요. 그리고 그림이 필요한 자리는 별도 컬럼에 "필요" 마커를 남깁니다.
>
> 산출물은 다섯 가지입니다. 가장 중요한 건 ③번 변환 결과 엑셀이고, 그 다음으로 ④번 manual_fixes.py — 60문항분 보완 데이터가 이 안에 패턴화돼 있어서, 다음 회차에도 30~50%는 그대로 복사해서 쓸 수 있다는 게 핵심입니다.

## 슬라이드 3 — 엑셀 양식 v2 컬럼 구조

> 화면: 17개 컬럼 표 (단순 양식 기준)

**멘트**
> 양식 자체는 17개 콘텐츠 컬럼이에요. 핵심 설계 결정이 두 가지입니다.
>
> 첫째, **그림은 무조건 별도 컬럼**입니다. 발문그림·보기①그림~보기④그림·해설그림·학습포인트그림. 보기가 그림형일 때도 따로 빼둬서, 위치가 깨지지 않게 했습니다.
>
> 둘째, **모든 텍스트 컬럼은 HTML 허용**입니다. `<sub>`, `<sup>`로 첨자·지수 표현하고, 표는 `<table>`, 오답분석은 `<ol><li>` 식으로 작성하면 업로드 시 그대로 렌더링됩니다.

## 슬라이드 4 — PDF에서 만난 이슈 15가지

> 화면: 세 카테고리 박스 (구조 6개 / 텍스트 5개 / 시각·파싱 4개)

**멘트**
> PDF에서 텍스트만 뽑으면 다 될 줄 알았는데, 실제로는 이슈가 15개 나왔습니다. 세 카테고리로 나눠봤어요.
>
> **구조 이슈**는 페이지 단위로 들어오는 노이즈예요. 페이지 1이 이전 회차 마지막 문항이라든지, 페이지 헤더가 매번 섞여 들어온다든지. 정규식 필터로 처리했습니다. 특히 ⑥번이 어이없는데, **PDF 마스터 데이터 자체에 문항 끝 태그가 다 "[전기이론]"으로 잘못 박혀** 있었어요. 그래서 카테고리는 페이지 안의 헤더 ("전기 기기", "전기 설비")로 다시 잡았습니다.
>
> **텍스트 이슈**는 더 까다롭습니다. 가장 큰 게 ⑦·⑧번 — 수식이 폰트 임베딩 문제로 통째로 빈 줄로 나오거나, 변수 글자가 빠집니다. ⑨번 한글 자모 공백 분리는 "①기 본 파" 같은 거고요.
>
> **시각·파싱 이슈**는 그림이나 표, 그리고 9번 같은 빈칸문제처럼 텍스트만으로는 어쩔 수 없는 부분입니다.

## 슬라이드 5 — 자동 변환 파이프라인

> 화면: 5단계 가로 흐름도 + 하단 결과 박스

**멘트**
> 변환은 5단계입니다.
>
> 1단계 pypdf로 페이지별 텍스트 뽑고, 2단계에서 노이즈 필터, 3단계에서 문항 번호 기준으로 발문·보기·해설·오답분석을 나눕니다. 4단계는 자동 보정인데, 자모 공백 정리하고 학습 POINT 분리하고 그림 마커 다는 단계예요. 5단계가 manual_fixes.py로 수식·표를 HTML로 덮어쓰는 단계입니다.
>
> 결과는 하단 박스에 있는 대로 60문항 100% 파싱, 자동 정상 변환 59건, 학습 POINT는 60문항 모두 채웠습니다. 남은 1건은 9번 빈칸문제인데, 이건 데이터는 정상이고 "이 문항은 빈칸 구조다"라는 노트만 남은 거라 사실상 0건이에요.

## 슬라이드 6 — 수식 HTML 변환 예시 (1번 콘덴서)

> 화면: 좌측 PDF 캡처 / 중간 추출 깨짐 / 우측 HTML 보완

**멘트**
> 어떻게 보완했는지 1번 문항으로 보여드리면 — 좌측이 원본 PDF 페이지 2의 1번 문제예요. 콘덴서 정전용량 문항인데, 해설에 보면 변수 C, S, d가 들어가 있죠.
>
> 가운데가 pypdf로 뽑은 결과인데, 변수 부분이 통째로 빠져서 `정전용량(    )은 극판의 넓이(    )에 비례` 이렇게 나옵니다. 이걸 우리 스크립트는 `[수식]` 마커로 표시해서 사람이 찾기 쉽게 했고요.
>
> 우측이 보완된 HTML입니다. `<i>C</i>`, `<i>S</i>`처럼 이탤릭 변수로 살리고, 학습 POINT는 `<p><strong>정전용량 공식</strong></p>` 헤더에 `<ul><li>` 리스트로 변수 설명까지 정리했습니다. 이걸 업로드하면 우리 시스템에서 그대로 수식 형태로 보입니다.

## 슬라이드 7 — 표 → HTML 표 변환 (23번)

> 화면: 좌측 PDF 캡처 (반작용 표 2개) / 우측 추출 깨짐 + HTML 재구성

**멘트**
> 표 케이스도 비슷합니다. 23번이 동기전동기 반작용 문제인데, PDF 원본에 표가 두 개 들어가 있어요.
>
> pypdf로 뽑으면 셀 경계가 다 사라지고 그냥 한 줄로 흘러나옵니다. "전류 위상 부하 성분 작용 명칭 현상…" 이런 식으로요. 이건 자동으로는 표로 복원 못 합니다.
>
> 그래서 우측 아래 보이는 것처럼 `<table>` 태그로 직접 재구성했어요. 23번뿐 아니라 30번 정류 맥동 주파수, 43번 매설깊이, 53번 완금 길이 같은 표 문항은 다 같은 방식으로 작성됐습니다.

## 슬라이드 8 — 그림 필요 영역 마커 (51번)

> 화면: 좌측 PDF 51번 그림기호 문제 / 우측 엑셀 51번 행 발췌

**멘트**
> 그림이 들어가야 하는 문항은 51번이 대표적이에요. "아래 그림기호가 나타내는 것은?" 묻는 문제인데, 정작 그림 자체는 PDF에서 텍스트로 추출이 안 됩니다.
>
> 그래서 발문그림 컬럼에 `[필요: 기호심볼 — 수동조작 접점]` 이렇게 마커를 명시했어요. 운영자가 엑셀을 봤을 때 어느 컬럼에 어떤 그림이 들어가야 하는지 한눈에 보이도록.
>
> 작업 흐름은 — 운영자가 이 마커를 보고 그림 파일을 업로드한 다음, 컬럼 값을 마커에서 `q51_manual_contact.png` 같은 실제 파일명으로 교체하면 됩니다. 이번 회차에서는 51번 외에 39번 TRIAC 심볼, 두 군데입니다.

## 슬라이드 9 — 학습 POINT 자동 분리 로직

> 화면: 좌측 분리 전 / 화살표 / 우측 분리 후 2컬럼

**멘트**
> PDF에는 사실 "학습 POINT"라는 별도 영역이 없어요. 그 대신 해설 본문 끝에 "공식", "법칙", "특징" 같은 박스가 붙어 있는데, 우리 양식에서는 이걸 별도 컬럼으로 빼고 싶습니다.
>
> 그래서 정규식으로 "OO 공식", "OO 법칙", "OO 특징" 같은 짧은 헤더 라인을 찾아서, 그 라인부터 끝까지를 학습 POINT 컬럼으로 자동 분리했습니다. 60문항 중 14건이 이 자동 분리로 잡혔고, 나머지는 수동 보완으로 채웠어요.
>
> 우측 보시면 분리 후 해설 컬럼은 짧은 설명만 남고, 학습 POINT 컬럼은 `<p><strong>패러데이의 법칙</strong></p>` 헤더 달린 HTML 블록이 들어갑니다. 우리 UI에서 두 영역을 다른 디자인으로 보여줄 수 있게 됐어요.

## 예상 질문 / 답변 준비

**Q1. 추정값 6문항은 어떻게 찾나요?**
A. 엑셀에서 발문 컬럼에 `<em>(원본 수치 PDF에서 추출 불가 — 일반 예제값으로 표기)</em>` 태그가 있는 행을 찾으면 됩니다. 또는 manual_fixes.py 파일에서 `원본 확인` 검색.

**Q2. 다른 회차 PDF는 어디 있나요?**
A. 현재 `data/templates/sample/2022년 1회.pdf`만 있습니다. 다음 회차 받으면 같은 폴더에 넣고 스크립트 회차별로 복사해서 실행.

**Q3. PDF가 안 바뀐 경우, 같은 작업 또 해야 하나요?**
A. 아니요. manual_fixes.py가 한 번 만들어지면 재실행만 하면 같은 결과 나옵니다. 코드 통째로 재사용 가능.

**Q4. HTML 안에 `<script>` 같은 거 들어가면 어떡하죠?**
A. 임포터 단계에서 sanitize 필수 (DOMPurify 또는 비슷한 라이브러리). 다음 단계 C에 포함.

**Q5. 9번 빈칸문제는 어떻게 보여줄 건가요?**
A. 발문에 `( ① )` `( ② )` 자리표시자가 그대로 들어가 있고, 보기에 `① 자속, ② 권수` 형태로 두 답을 묶어 표기. UI에서 정답 채점은 기존 1·2·3·4 로직 그대로 사용 가능합니다.
