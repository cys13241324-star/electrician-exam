왜 박스(틀)는 class가 아니라 inline style로 써야 하나

운영자가 엑셀 셀에 쓴 내용은 런타임 데이터라, Tailwind가 빌드 때 스캔하지 못합니다. 그래서 셀에 적은 class 이름은 CSS가 만들어지지 않습니다. 이 페이지는 그 상황(앱 CSS 없음)을 그대로 재현한 것입니다 — 운영 화면에서도 동일하게 발생합니다.

방식 ① class 로 박스를 그린 경우

엑셀 셀 내용: <div class="rounded-xl border-l-4 border-emerald-600 bg-emerald-50 p-4"> … </div>

핵심 공식
옴의 법칙: V = I R
전력: P = V I = I² R

→ 결과: 박스가 사라지고 그냥 글자만. 색·테두리·여백·라운드 전부 무시됨. "핵심 공식"이 일반 문장에 묻혀 강조 효과가 없어집니다.

방식 ② inline style 로 박스를 그린 경우

엑셀 셀 내용: <div style="border:1px solid #34d399;background:#ecfdf5;…"> … </div>

핵심 공식
옴의 법칙: V = I R
전력: P = V I = I² R
주의
단위를 V 가 아니라 mV 로 두면 1000배 오차.

→ 결과: CSS 환경과 무관하게 박스가 그대로. 요소가 자기 모양을 들고 다니기 때문. 관리자 미리보기·일괄등록 미리보기·PPT·다른 앱/내보내기 어디서든 동일.

정리

  • class 방식 = 모양 정의가 앱 빌드에 의존 → 엑셀에 쓴 클래스는 Tailwind가 못 봐서 CSS가 안 생김 → 박스 안 나옴.
  • inline style 방식 = 모양이 콘텐츠에 같이 붙어 다님 → 어디서나 동일하게 렌더.
  • 그래서 학습POINT 안의 강조 박스는 작성규칙서의 표준 inline 스니펫(3-A·3-B)만 복사해 쓴다.

※ 독끝 학습 POINT의 바깥 초록 박스는 앱 코드(.tsx)가 직접 그리므로 정상입니다. 문제는 운영자가 콘텐츠 안에서 박스를 만들 때만 발생합니다.
