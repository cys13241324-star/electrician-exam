# CBT 문항 콘텐츠 작성 규칙서 (v1 초안)

> 대상 셀: **발문 · 해설 · 오답분석 · 학습포인트**
> ⚙️ 이 문서는 **작업용 초안**입니다. 만들며 우리가 조정·수정하고, 모든 게 확정되면 그 상태를 최종 규칙으로 고정합니다.
> 이 규칙은 운영자가 엑셀 셀에 글을 쓸 때 지켜야 하는 "코드 양식"입니다.
> 화면에는 셀 내용이 **HTML 그대로** 표시되고, `$…$` 안의 수식만 KaTeX로 변환됩니다.

---

## 0. 큰 원칙 3가지

1. **HTML은 최소한만.** 구조를 만드는 태그(목록·표)는 쓰지 않는다.
2. **수식·기호는 LaTeX로.** `$ … $`(인라인) / `$$ … $$`(독립 줄).
3. **박스(틀)가 필요하면 inline style로 직접 그린다.** 앱 디자인에 기대지 않는다.

---

## 1. 허용 HTML 태그 (화이트리스트)

이 목록에 **없는 태그는 쓰지 않는다.**

| 용도 | 허용 태그 |
|---|---|
| 강조 | `<strong>` `<em>` `<u>` |
| 위/아래첨자 | `<sub>` `<sup>` |
| 줄바꿈 | `<br>` |
| 그림 | `<img src="파일명">` |
| 박스(틀) | `<div style="…">` — 3번 항목 규격대로만 |

> `<` `>` `&` 자체를 글자로 보여줘야 할 땐 `&lt;` `&gt;` `&amp;`.

---

## 2. 금지 태그와 대체 방법 (★ 이사님 지시 반영)

| 금지 | 이유 | 대신 이렇게 |
|---|---|---|
| `<ol> <ul> <li>` | 구조 태그 금지 | 줄마다 `<br>` + 번호를 **글자로** 직접 (번호 표기 방식은 이사님 확정 예정) |
| `<table> <tr> <td>` 등 | 구조 태그 금지 | 표·행렬은 LaTeX `$$\begin{array}{…}\end{array}$$` |
| `<p>` | 불필요 | 문단 구분은 `<br><br>` |
| `<code> <pre>` | 수식 렌더 차단됨 | 수식은 LaTeX, 그 외는 일반 텍스트 |
| `class="…"` | 앱 CSS 의존 → 깨짐 | 모양은 `style="…"` inline으로 |

### 번호 목록 예시 (오답분석에서 자주 씀)

> 번호 **표기 방식(①②③ / 1) / (1) 등)은 이사님이 확정**합니다.
> 아래는 "`<ol>` 대신 `<br>`로 줄을 나눈다"는 구조만 보여주는 예시입니다.

```
❌ <ol><li>보기1은 …</li><li>보기2는 …</li></ol>

✅ (번호) 보기1은 단위가 달라 오답.<br>
   (번호) 보기2는 공식 $P=VI$ 를 $P=V/I$ 로 잘못 적용.<br>
   (번호) 보기4는 부호 반대.
```

### 표 예시 (LaTeX array)

```
✅ $$\begin{array}{c|c}
   소자 & 기호 \\ \hline
   저항 & R \\
   콘덴서 & C
   \end{array}$$
```

> ⚠️ KaTeX 한계: `\begin{array}` `\begin{matrix}` `\begin{cases}` `\begin{aligned}` 는 됨.
> `\begin{enumerate}` `\begin{itemize}` `\begin{tabular}` 는 **안 됨**(렌더 실패).
> → 목록은 반드시 `<br>`+번호 글자 방식으로.

## 3. 박스(틀) — inline style 규격 ★ 이사님 지시 반영

독끝 학습 POINT의 **바깥 초록 박스는 시스템이 자동으로 그립니다. 운영자는 손대지 않습니다.**

학습포인트(또는 해설) **안에 작은 강조 박스**(핵심공식·주의 등)를 넣고 싶을 때만,
아래 **표준 스니펫을 복사**해서 내용만 바꿔 씁니다. (색은 hex로 박아 둬서 어디서든 똑같이 나옴)

### 3-A. 핵심 공식 박스 (초록 계열 — 학습POINT 톤과 통일)

```html
<div style="border:1px solid #34d399;background:#ecfdf5;border-radius:8px;padding:10px 12px;margin:8px 0;">
<strong style="color:#047857;">핵심 공식</strong><br>
$$V = I R$$
</div>
```

### 3-B. 주의 박스 (호박색 — 함정·실수 경고용)

```html
<div style="border:1px solid #fbbf24;background:#fffbeb;border-radius:8px;padding:10px 12px;margin:8px 0;">
<strong style="color:#b45309;">주의</strong><br>
단위를 V 가 아니라 mV 로 두면 1000배 오차.
</div>
```

**규칙**
- `style` 안의 값(테두리·배경·여백·라운드)은 **임의로 바꾸지 않는다.** 위 두 종류만 사용.
- 박스 안에서도 1·2번 규칙(허용 태그, LaTeX)은 동일하게 적용.
- 박스를 중첩하지 않는다(박스 안 박스 금지).

## 4. 그림

- 본문 어디에나 `<img src="파일명">`. 여러 개·텍스트와 번갈아 OK.
- 파일이 아직 없으면 `<img src="[필요: 무슨 그림인지 설명]">` 로 표시.

### 4-1. 이미지 파일명 규칙 ★ 이사님 지시 반영

문항코드가 규칙으로 생성되듯, 이미지 파일명도 **문항코드에서 자동으로 유도**한다.

**형식:** `{문항코드}_{슬롯}{순번}.{확장자}`

- `{문항코드}` — 그 문항의 코드 그대로 (예: `elec_A_2022_01_21`)
- `{슬롯}` — 그림이 들어가는 자리. **ASCII 토큰** 사용 (아래 표)
- `{순번}` — 같은 슬롯에 그림이 2장 이상일 때만 `1,2,3…`. 1장뿐이면 생략 가능하나, **붙이는 걸 권장**(나중에 추가돼도 규칙 안 깨짐)
- `{확장자}` — 소문자 `.png` 권장 (사진류는 `.jpg`)

| 그림 위치 | 슬롯 토큰 | 예시 파일명 |
|---|---|---|
| 발문 그림 | `stem` | `elec_A_2022_01_21_stem1.png` |
| 보기 1~4 | `opt1`~`opt4` | `elec_A_2022_01_21_opt1.png` |
| 해설 그림 | `exp` | `elec_A_2022_01_21_exp1.png` |
| 오답분석 그림 | `wrong` | `elec_A_2022_01_21_wrong1.png` |
| 학습포인트 그림 | `lp` | `elec_A_2022_01_21_lp1.png` |

> 슬롯 토큰을 **영어로 확정**. 이유: 업로드 서버가 한글·특수문자를 `_`로 치환하므로
> 한글("보기")은 파일명에 살아남지 못함. ASCII 토큰이라야 깨지지 않고 URL에도 안전.

**셀에서 참조할 때도 같은 이름을 그대로:**

```
<img src="elec_A_2022_01_21_exp1.png">
```

> ※ 작업 방향: 현재 업로드 API는 `{코드}__{슬롯}__{원본이름}` 으로 자동 저장 중.
> → **이 규칙에 맞게 업로드 코드를 수정해 통일한다** (우리가 만들며 조정 → 최종 확정 시 규칙 고정).
> 기존 업로드 파일은 옛 이름 유지(참조 안 깨짐), 새 업로드부터 새 규칙 적용.

## 5. 영역별 적용 요약

| 영역 | 핵심 |
|---|---|
| 발문 | 일반 텍스트 + 필요한 수식 `$…$`. 목록 거의 불필요. |
| 해설 | 설명 텍스트 + `$$…$$` 공식 + `<img>`. 강조는 핵심공식 박스(3-A). |
| 오답분석 | 보기별로 `①②③④` + `<br>`. **`<ol>` 절대 금지.** |
| 학습포인트 | 핵심 정리. 공식은 3-A 박스, 함정은 3-B 박스. |

## 6. 제출 전 셀프 체크리스트

- [ ] `<ol> <ul> <li> <table> <p> <code> <pre>` 안 썼나?
- [ ] `class=` 안 썼나? 모양은 전부 `style=` 인가?
- [ ] 박스는 3-A / 3-B 표준 그대로인가? (값 변경·중첩 없음)
- [ ] 수식은 전부 `$…$` / `$$…$$` 안에 있나?
- [ ] 표·행렬을 `<table>` 아니라 `$$\begin{array}$$` 로 했나?
- [ ] `<` `>` `&` 를 글자로 쓸 땐 `&lt; &gt; &amp;` 인가?
- [ ] 이미지 파일명이 `{문항코드}_{슬롯}{순번}.png` 규칙(4-1)을 따르나?
