CBT 문항 작성 규칙 — 정리 (v1 · 현재 확정 상태)

대상 셀: 발문 · 보기 · 해설 · 오답분석 · 학습포인트. 화면에는 셀 내용이 HTML 그대로 표시되고, $…$ 안의 수식만 KaTeX로 변환됩니다. 작업용 초안 — 확정 시 정식 양식(build_template_v3.py)에 반영.

① 큰 원칙 3가지

  1. HTML은 최소한만. 구조 만드는 태그(목록·표·문단)는 쓰지 않는다.
  2. 수식·기호는 LaTeX로. $ … $(인라인) / $$ … $$(독립 줄).
  3. 박스(틀)가 필요하면 inline style로 직접. 앱 디자인(class)에 기대지 않는다.

② 문항코드 규칙

형식elec_<교재구분>_<연도>_<회차2자리>_<번호2자리>
예시elec_A_2022_01_01  (교재 A · 2022년 · 1회 · 1번)
자동화엑셀 작성행은 수식으로, 출처(교재구분·연도·회차·번호)만 채우면 자동 표시. 샘플 행은 정적값. 제출 후 서버가 출처로 다시 계산하므로 수식이 미계산이어도 안전.

②-2 과목ID (전기기능사 한정 — 3종 고정)

과목 (한글)과목ID 입력값 (영문)설명
전기이론theory전기·자기 이론, 회로 해석
전기기기machinery발전기·전동기·변압기·정류 등
전기설비facility옥내배선·전기기구·시설 규정
전기기능사는 위 3가지만 사용한다. 「과목ID」 셀에는 반드시 영문 입력값(theory / machinery / facility)을 적는다. 한글(전기이론 등)을 그대로 넣지 않는다.

③ 이미지 파일명 규칙

형식{문항코드}_{슬롯}{순번}.png  (소문자·영문, 공백 없음)
방식작성자가 이 규칙대로 파일명을 지어서 저장한 뒤 업로드한다. 본문에서는 같은 이름으로 <img src="…"> 참조. (서버 자동 부여 여부와 무관하게, 작성자가 규칙명으로 올린다.)
슬롯 토큰발문 stem · 보기 opt1~opt4 · 해설 exp · 오답분석 wrong · 학습포인트 lp  (한글은 깨져서 영문 확정)
순번같은 슬롯에 그림이 여러 장이면 1, 2, 3 …. 보기는 슬롯에 번호가 있으므로(opt1~4) 보통 순번 생략.

예시 — 문항코드가 elec_A_2022_01_13 인 경우

그림 위치파일명본문 참조
발문 그림elec_A_2022_01_13_stem1.png<img src="elec_A_2022_01_13_stem1.png">
보기 2 그림elec_A_2022_01_13_opt2.png<img src="elec_A_2022_01_13_opt2.png">
해설 첫째 그림elec_A_2022_01_13_exp1.png해설 셀: …설명… <img src="elec_A_2022_01_13_exp1.png"> …
해설 둘째 그림elec_A_2022_01_13_exp2.png<img src="elec_A_2022_01_13_exp2.png">
오답분석 그림elec_A_2022_01_13_wrong1.png<img src="elec_A_2022_01_13_wrong1.png">
학습포인트 그림elec_A_2022_01_13_lp1.png<img src="elec_A_2022_01_13_lp1.png">
파일명을 임의로 짓지 말 것. 위 규칙({문항코드}_{슬롯}{순번}.png)을 정확히 지켜 저장·업로드해야 본문 <img src> 가 그대로 연결된다. 그림이 아직 없으면 본문에 <img src="[필요: 설명]"> 마커.

오답분석 예시 (원문자)

① 보기1은 단위가 달라 오답.<br>
② 보기2는 공식 $P=VI$ 를 $P=V/I$ 로 잘못 적용.<br>
③ 보기4는 부호 반대.

⑥ 수식 — LaTeX (KaTeX)

인라인$ ... $   예) 저항 $R=\dfrac{V}{I}$
디스플레이$$ ... $$   예) $$P=VI=I^2 R$$
가능 환경array · matrix · cases · aligned
불가 환경enumerate · itemize · tabular → 목록은 <br>+원문자
⚠️ KaTeX는 수식 환경만 지원. 번호 목록을 LaTeX로 만들 수 없으므로 ⑤의 원문자 방식 사용.

⑦ 박스 — inline style 표준 스니펫 (독끝 학습 Point · 회색)

학습포인트/해설 안에 강조 박스가 필요하면 아래 「독끝 학습 Point」 회색 박스 하나로 통일한다. 복사해서 내용만 교체. style 값 변경·박스 중첩 금지. (이전의 핵심공식/주의 2종은 폐기)

표준 스니펫 — 독끝 학습 Point (회색)

<div style="border:1px solid #cbd5e1;background:#f1f5f9;border-radius:8px;padding:10px 14px;margin:8px 0;">
<strong style="color:#475569;">독끝 학습 Point</strong><br>
$$C=\dfrac{\varepsilon S}{d}\;[\text{F}]$$
</div>

→ 실제 렌더 (수식 자리는 KaTeX로 변환됨):

독끝 학습 Point
C = εS / d  [F]

텍스트형 예 (수식 없이도 동일 박스 사용):

독끝 학습 Point
단위를 V 가 아니라 mV 로 두면 1000배 오차.

⑧ 제출 전 셀프 체크리스트

  • <ol> <ul> <li> <table> <p> 안 썼나?
  • class= 안 썼나? 모양은 전부 style= 인가?
  • 번호는 원문자 ①②③ 인가?
  • 표를 <table> 아니라 $$\begin{array}$$ 로 했나?
  • 수식은 전부 $…$ / $$…$$ 안에 있나?
  • 박스는 「독끝 학습 Point」 회색 표준 그대로인가? (값 변경·중첩 없음)
