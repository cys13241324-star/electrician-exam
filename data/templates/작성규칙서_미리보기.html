CBT 문항 작성 규칙 — 정리 (v1 · 현재 확정 상태)

대상 셀: 발문 · 보기 · 해설 · 오답분석 · 학습포인트. 화면에는 셀 내용이 HTML 그대로 표시되고, $…$ 안의 수식만 KaTeX로 변환됩니다. 작업용 초안 — 확정 시 정식 양식(build_template_v3.py)에 반영.

① 큰 원칙 3가지

  1. HTML은 최소한만. 구조 만드는 태그(목록·표·문단)는 쓰지 않는다.
  2. 수식·기호는 LaTeX로. $ … $(인라인) / $$ … $$(독립 줄).
  3. 박스(틀)가 필요하면 inline style로 직접. 앱 디자인(class)에 기대지 않는다.

② 문항코드 규칙

형식elec_<교재구분>_<연도>_<회차2자리>_<번호2자리>
예시elec_A_2022_01_01  (교재 A · 2022년 · 1회 · 1번)
자동화엑셀 작성행은 수식으로, 출처(교재구분·연도·회차·번호)만 채우면 자동 표시. 샘플 행은 정적값. 제출 후 서버가 출처로 다시 계산하므로 수식이 미계산이어도 안전.

③ 이미지 파일명 규칙

형식{문항코드}_{슬롯}{순번}.png
슬롯 토큰발문 stem · 보기 opt1~opt4 · 해설 exp · 오답분석 wrong · 학습포인트 lp
예시elec_A_2022_01_39_lp1.png (39번 학습포인트 1번 그림)
부여 주체그림 업로드 시 서버가 자동 부여(엑셀에서 자동 생성하지 않음). 같은 슬롯 재업로드 시 같은 이름으로 덮어써서 파일명이 항상 고정.
참조본문 어디에나 <img src="elec_A_2022_01_39_lp1.png">. 슬롯 토큰은 한글이 깨져 영어 확정.

⑤ 금지 태그와 대체 방법

금지대신 이렇게
<ol> <ul> <li>줄마다 <br>, 번호는 원문자 ①②③④… 로 작성자가 직접 입력
<table> <tr> <td>표·행렬은 LaTeX $$\begin{array}{…}\end{array}$$
<p>문단 구분은 <br><br>
class="…"모양은 전부 style="…" inline (⑦)

오답분석 예시 (원문자)

① 보기1은 단위가 달라 오답.<br>
② 보기2는 공식 $P=VI$ 를 $P=V/I$ 로 잘못 적용.<br>
③ 보기4는 부호 반대.

⑥ 수식 — LaTeX (KaTeX)

인라인$ ... $   예) 저항 $R=\dfrac{V}{I}$
디스플레이$$ ... $$   예) $$P=VI=I^2 R$$
가능 환경array · matrix · cases · aligned
불가 환경enumerate · itemize · tabular → 목록은 <br>+원문자
⚠️ KaTeX는 수식 환경만 지원. 번호 목록을 LaTeX로 만들 수 없으므로 ⑤의 원문자 방식 사용.

⑦ 박스 — inline style 표준 스니펫

독끝 학습 POINT의 바깥 초록 박스는 앱이 자동으로 그림(셀에 쓰지 않음). 아래는 학습포인트/해설 안에 넣는 작은 박스. 복사해서 내용만 교체. style 값 변경·박스 중첩 금지.

7-A. 핵심 공식 박스 (초록)

<div style="border:1px solid #34d399;background:#f0fdf4;border-radius:8px;padding:10px 14px;margin:8px 0;">
$$C=\dfrac{\varepsilon S}{d}\;[\text{F}]$$
</div>

→ 실제 렌더 (수식 자리는 KaTeX로 변환됨):

핵심 공식
C = εS / d  [F]

7-B. 주의 박스 (호박색)

<div style="border:1px solid #fbbf24;background:#fffbeb;border-radius:8px;padding:10px 14px;margin:8px 0;">
<strong style="color:#b45309;">주의</strong><br>
단위를 V 가 아니라 mV 로 두면 1000배 오차.
</div>

→ 실제 렌더:

주의
단위를 V 가 아니라 mV 로 두면 1000배 오차.

⑧ 제출 전 셀프 체크리스트

  • <ol> <ul> <li> <table> <p> 안 썼나?
  • class= 안 썼나? 모양은 전부 style= 인가?
  • 번호는 원문자 ①②③ 인가?
  • 표를 <table> 아니라 $$\begin{array}$$ 로 했나?
  • 수식은 전부 $…$ / $$…$$ 안에 있나?
  • 박스는 7-A / 7-B 표준 그대로인가? (값 변경·중첩 없음)
  • 그림 참조가 {문항코드}_{슬롯}{순번}.png 형식인가?
