# 작성규칙서에 대한 코멘트 (보고용 메모)

> 같이 볼 것: `작성규칙서_v1초안.md`(규칙 본문), `inline_style_데모.html`(브라우저로 열어 ①② 비교)
> 이 글은 "왜 이렇게 정했나 / 무엇을 짚어야 하나"를 정리한 코멘트입니다. 규칙 본문은 작업하며 계속 다듬는 초안입니다.

## 1. 코드 양식(작성 규칙) — 왜 만드나

문항코드가 규칙으로 자동 생성되듯, **셀에 들어가는 내용도 사람마다 제각각이면 시스템이 못 받습니다.**
화면은 셀 내용을 거의 그대로 HTML로 표시하므로, 운영자가 쓰는 문법을 고정해야 품질·안정성이 잡힙니다.
→ 허용 태그 화이트리스트 + 금지 태그 + 표준 예시 + 제출 전 체크리스트로 정리했습니다.

## 2. `<ol>` 금지 → LaTeX : 짚을 점 (중요)

이사님 지시는 "구조 태그(`<ol>` 등) 빼고 LaTeX로". 규칙엔 반영했는데 **기술적 사실 하나를 꼭 알고 결정하셔야 합니다.**

- 수식 엔진은 KaTeX이고, KaTeX는 **`$…$` 안의 수식만** 처리합니다.
- LaTeX의 목록 명령(`\begin{enumerate}`)·표 명령(`\begin{tabular}`)은 **렌더되지 않습니다.**
- 즉 "번호 목록을 LaTeX로" 는 글자 그대로는 불가능합니다.
- 그래서 현실적 대체안으로: **번호 목록 = `<br>` + 번호 글자**, **표 = `$$\begin{array}…$$`**(이건 KaTeX가 지원).
- **번호 표기 방식(①②③ / 1) / (1) …)은 이사님이 정하실 부분**이라 규칙엔 비워뒀습니다.

→ 코멘트: 엔진을 바꾸지 않는 한 목록은 HTML도 LaTeX도 아닌 "줄바꿈+번호글자"가 최선입니다. 이 절충이 의도와 맞는지 확인 부탁드립니다.

## 3. 박스는 inline style : 왜 그런가 (데모로 확인)

가장 오해하기 쉬운 부분이라 데모(`inline_style_데모.html`)를 만들었습니다. 핵심:

- 이 프로젝트는 Tailwind CSS. **Tailwind는 빌드 때 소스코드(.tsx)를 훑어 거기 있는 클래스만 CSS로 만듭니다.**
- 운영자가 엑셀 셀에 쓴 내용은 런타임 데이터라 Tailwind가 못 봅니다.
- 그래서 셀에 `class="...박스..."` 라고 써도 **그 CSS가 아예 안 만들어져 → 실서비스에서 박스가 안 나옵니다.** (다른 화면이 아니라 본 서비스에서 이미 깨짐)
- `inline style`(`style="..."`)은 요소가 모양을 직접 들고 다녀 **CSS 환경과 무관하게 어디서나 동일**하게 그려집니다.

→ 코멘트: 그래서 "콘텐츠 안에서 박스가 필요하면 inline style로, 틀을 잡고 써야 한다"는 지시가 기술적으로 정확합니다. 운영자 임의 변형을 막기 위해 **표준 스니펫 2종(핵심공식·주의)만** 복붙하도록 규정했습니다. 독끝 학습 POINT의 **바깥 초록 박스는 앱 코드가 그리므로 운영자는 안 건드립니다.**

## 4. 이미지 파일명 규칙 — 영어 토큰으로 확정한 이유

- 형식: `{문항코드}_{슬롯}{순번}.png` (예: `elec_A_2022_01_21_exp1.png`)
- 이사님 예시는 `코드_보기1`(한글)이었으나, **업로드 서버가 한글·특수문자를 `_`로 치환**합니다(파일시스템·URL 안전 때문). 한글 "보기"는 파일명에 못 살아남습니다.
- 그래서 의미는 동일하게 두되 슬롯 토큰을 **영어로 확정**: 발문=stem, 보기=opt1~4, 해설=exp, 오답분석=wrong, 학습포인트=lp.
- 업로드 코드(`/api/upload/route.ts`)를 이 규칙대로 **이미 수정 완료**. 기존 업로드 파일은 옛 이름 유지(참조 안 깨짐), 새 업로드부터 새 규칙 적용.

→ 코멘트: 이미지 이름이 문항코드에서 자동 유도되므로, 어떤 그림이 어느 문항·어느 자리 것인지 파일명만 봐도 식별됩니다.

## 5. 작업 원칙 (확정)

규칙서는 **못 박는 문서가 아니라 작업용 초안**입니다. 만들며 우리가 조정·수정하고,
모든 게 확정되면 그 상태를 최종 규칙으로 고정 → `tools/build_template_v3.py`의
사용가이드·색상범례 시트에 반영하고 v3 엑셀 양식을 재생성합니다.

## 6. 이사님 회신 부탁 항목

1. 2번 절충안(목록=줄바꿈+번호글자, 표=LaTeX array)이 의도와 맞는지
2. 번호 표기 방식 (①②③ / 1) / (1) 중 무엇)
3. 박스 표준 스니펫의 색·종류(현재 핵심공식=초록, 주의=호박색) 이대로 OK인지
4. 추가로 규칙에 넣었으면 하는 항목
